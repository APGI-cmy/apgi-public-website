## Purpose

This addendum aligns the APW App Description with the root website intent canon.

The existing App Description remains authoritative for the corrective multi-page public website baseline. This addendum strengthens the strategic product narrative so the website is not implemented as a generic security consultancy brochure, a services catalogue, or a flat APGI Hub card directory.

---

## App description alignment

APW must describe APGI as a holistic loss-prevention and maturity-improvement partner.

The public website must explain that APGI helps organisations prevent loss by improving operational maturity. Loss is framed as arising from opportunity, and opportunity is generated when organisational environments are immature or weak across governance, process integrity, people, systems, controls, data, risk management, and assurance.

APW must make both loss categories understandable:

- **Malicious loss:** theft, fraud, collusion, deliberate wrongdoing, manipulation, and exploitation of control weaknesses.
- **Non-malicious loss:** operational failure, weak process integrity, poor maintenance, process variance, misaligned procedures, weak data, human error, and uncontrolled change.

---

## Maturity journey requirement

The maturity journey is the conceptual spine of the APW website.

Visitors must understand that APGI helps organisations:

1. assess where they are;
2. understand their current maturity level;
3. identify gaps and opportunity generators;
4. build a structured maturity roadmap;
5. implement the roadmap through consulting, tools, monitoring, coaching, and training;
6. mature toward resilient operations where controls are embedded, measured, monitored, and increasingly automated.

The Platform/APGI Hub route must therefore explain the ecosystem as one maturity journey, not as disconnected products.

---

## Free assessment requirement

The App Description is amended to require a public free maturity assessment entry point.

The free assessment is an introductory maturity indication, not a full audit or full assessment engine inside APW. APW should introduce it clearly, explain its purpose, and hand the visitor off to the ISMS/APGI Hub destination where deeper assessment functionality belongs.

---

## Ecosystem narrative requirement

APW must explain ecosystem roles in the maturity journey:

- MMM / Maturity Roadmap: assessment, criteria, maturity level, gaps, and roadmap.
- Risk Management: risk logic, controls, residual risk, and mitigation planning.
- PIT: long-term roadmap implementation tracking.
- Incident & Intelligence Hub: workflow for incidents, anomalies, weak signals, and response.
- RADAM / Data Analytics / Remote Assurance: monitoring, control effectiveness, and independent assurance.
- Skills Development / Training: capability building for maturity progression.
- ISMS / APGI Hub: platform front door and handoff destination.

---

## Boundary

This addendum does not approve APW to implement backend assessment logic, Supabase storage, CRM, auth, LMS administration, or private platform functionality. APW introduces, explains, and hands off.
