## UX Requirement

The Training page must support two distinct discovery paths:

1. Classroom/contact-delivery catalogue discovery.
2. E-learning / Thinkific platform discovery.

The user must not have to infer that classroom offerings exist from one VPSHR card or a generic storefront button.

---

## Required Journey: Training Catalogue Discovery

| Step | User Action | System Response |
|---|---|---|
| 1 | User opens `/training`. | Page presents a hero explaining APGI training modes. |
| 2 | User scans delivery choices. | UI separates classroom/contact delivery from e-learning/platform delivery. |
| 3 | User reviews classroom offering cards. | Each card shows course name, audience, duration, delivery mode, and assessment indicators where available. |
| 4 | User selects an offering. | User is directed to contact APGI for classroom/group/custom delivery. |
| 5 | User reviews e-learning cards. | E-learning offerings show platform availability and link to Thinkific storefront. |

---

## UX Rules

- VPSHR may be featured, but must not dominate the page as the only meaningful training product.
- The catalogue must be scannable on mobile and desktop.
- Classroom and e-learning CTAs must be visually distinct.
- Thinkific handoff must remain public storefront only.
- No admin/LMS-management destinations may appear.
- If course-specific public URLs are unavailable, cards must use the approved Thinkific storefront or contact CTA.
