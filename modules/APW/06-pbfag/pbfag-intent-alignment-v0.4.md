## Purpose

This addendum updates the PBFAG readiness posture so APW implementation cannot proceed as aligned if the root website intent is missing.

---

## Additional PBFAG checks

Before any further APW implementation work is treated as ready, the Foreman/QP review must confirm:

1. `APW_WEBSITE_INTENT.md` is loaded and cited as a root alignment artifact.
2. Stage 1 through Stage 11 artifacts are checked for maturity/loss-prevention alignment.
3. The build target explains APGI's holistic loss-prevention proposition.
4. The build target includes malicious and non-malicious loss framing.
5. The build target presents the maturity journey as the conceptual spine.
6. The free maturity assessment entry point and ISMS/APGI Hub handoff are included or explicitly governed as pending.
7. Platform/APGI Hub content is journey-led, not merely a card grid.
8. Ecosystem roles are connected to the maturity journey.
9. Hover/click/ask/train interaction is treated as a learning pattern where used.
10. Case-study content is anonymised unless specific permission is recorded.

---

## PBFAG disposition rule

If the above checks fail, the gate remains RED or conditional. A working build, Vercel preview, or visually polished page is not enough if it loses the APGI maturity journey.

This addendum is a prebuild gate correction. It does not authorize new runtime scope inside APW.
