# APW Architecture v0.4 Trace Matrix

## Status

Drafted for review.

## Purpose

This trace matrix links TRS v0.4 and TRS v0.4A hardening requirements to Architecture v0.4 design decisions.

| TRS requirement group | Architecture coverage |
|---|---|
| Static public-site boundary | Architectural Boundary; Negative Architecture Requirements |
| Route model hardening | Route Architecture; Route Registry Architecture |
| Structured data schema | Route Registry Architecture; Content Data Architecture |
| Required component families | Component Architecture |
| Hover -> Click -> Ask -> Train | Interaction Architecture |
| Ask Maturion fallback | Interaction Architecture; Public-Safety Architecture |
| Dialog/drawer/detail accessibility | Accessibility Architecture |
| Static runtime and performance | Architectural Boundary; Component Architecture; Negative Architecture Requirements |
| Public-safety and content governance | Public-Safety Architecture |
| SEO, metadata, and route registry | Metadata and SEO Architecture |
| Testability and QA traceability | Validation Architecture |
| File-level implementation direction | File-Level Architecture Direction |

## Gate

This trace matrix does not authorize implementation.
